# Research

Everything else in this handbook is grounded in code that shipped. This section is the one place that
points *forward* — where agent systems are going, and the through-line that connects the rungs already
built to the ones the field is climbing now.

The rule here is the same as everywhere else, just harder to keep: **separate signal from hype.** A
forward-looking note is worthless if it's a press-release roundup. So each one states what's actually
load-bearing, what's still a bet, and — honestly — where this body of work is and isn't ready for what's
coming.

| Note | What it argues |
|------|----------------|
| [The autonomy-ladder thesis](the-autonomy-ladder-thesis.md) | Most "agent" debates are altitude confusion. There's a ladder from plain code to a swarm, and the engineering skill is climbing *only as high as the problem needs*. |
| [Where agent systems are heading](where-agent-systems-are-heading.md) | The reliability problems are mostly solved patterns now. The next frontier is agents talking to *each other*, agents with *identities*, cheaper *specialised* models, and agents that act *before* asked — and which of those is signal. |

---

### The bias of this section

Two years of building these systems taught one durable lesson, and it's the lens for everything here:
the hard part of an agent system is never the model call — it's the scaffolding around it. That bias
makes this section skeptical of any prediction whose payoff depends on the model simply getting smarter,
and interested in the predictions that are really *systems* problems wearing a new name. Most of what's
coming is the latter.
