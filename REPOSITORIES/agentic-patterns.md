# agentic-patterns

> Seven agent architecture patterns, each with runnable Node and Python starters.

## Problem

Patterns live in blog posts and slide decks where you can't run them. You can't feel the
difference between orchestrator-worker and a DAG by reading about it.

## Architecture

7 documented patterns (orchestration, routing, DAG scheduling, and more), each shipping a
runnable starter in both Node and Python. The DAG pattern uses Kahn's topological sort - the
deterministic backbone behind "[banned probabilistic control flow](../FIELD-NOTES/banned-probabilistic-control-flow.md)."

## Lessons learned

- A runnable pattern teaches in minutes what a diagram can't teach in an hour.
- Most production systems are 2–3 patterns composed, not one exotic architecture.

## Demo

Clone and run any pattern's starter in your language of choice.
